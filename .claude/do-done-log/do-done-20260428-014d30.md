## Plan

**1. Add a config detector in `stepper.lua`**
```lua
local function currentDisplayConfig()
  for _, s in ipairs(hs.screen.allScreens()) do
    if (s:name() or ""):find("Sidecar") then return "sidecar" end
  end
  return #hs.screen.allScreens() == 1 and "single" or "multi"
end
```
Purely additive — if no Sidecar screen, behavior is unchanged.

**2. ctrl+opt+arrow → "toggle to other screen" in sidecar mode** (`stepper.lua:957–1002`)
At the top of `moveToDisplay`, if config is `sidecar`, rewrite `position` to `"other"` and pick whichever screen isn't current. All 5 keys (←↑→↓ + return) hit the same code path, which means the existing 1-hour undo logic keeps working naturally — second press of any of them within TTL restores. screenmemory just keys under role `"other"` and uses proportional-with-edge-snap on first visit (same logic that already gives you "right-edge stays right-edge" on the regular external-monitor setup).

**3. fn+arrow shoves per-screen in sidecar mode** (`stepper.lua:85–104`)
Change the gate in `dispatchStepMove` and `withReset` from `#screens == 1` to `cfg == "single" or cfg == "sidecar"`. L010's `ofsr.shove` already operates on `win:screen()`, so per-screen shoving on sidecar is automatic — windows stay put and absorb into edges instead of jumping to the iPad.
