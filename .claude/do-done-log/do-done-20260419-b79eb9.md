## Current solution recap

We watch `AXSelectedTextChanged` at the Bear app level — we never look for `⌘V` specifically. On every fire we check: (1) did ￼-count go up by 1, (2) is there an image on clipboard, (3) did `hs.pasteboard.changeCount()` advance since our last fire. All three true → we write the width comment at the caret. Correct: we're **blind to "paste"** — we just detect "a new ￼ appeared AND the clipboard changed recently." The advantage you noted (Paste app, drag-and-drop, context-menu paste) is exactly why I picked observer over keystroke-interception.

## Why AX can't distinguish image from format-glyph

Bear uses macOS's standard `NSTextAttachment` for BOTH images and format glyphs (bullet dots, blockquote bars, HR lines, checkboxes). `NSTextAttachment` is always represented as **U+FFFC (￼)** in the text stream — that's literally what the character is called: "Object Replacement Character," a placeholder. AX shows us the text stream. It does not expose attachment metadata (no `AXAttachmentType`, no file URL, no glyph kind). We confirmed this with the probe script — no AX attribute I could find would differentiate. So from outside Bear: one flat wall of identical ￼. Surprising but consistent.

## AppleScript / SQLite route

Two options here:
- **Bear's URL scheme with `x-success`** callback (`bear://x-callback-url/open-note?id=X&x-success=hammerspoon://...`) can return the real markdown. Works, but heavyweight: each call roundtrips through macOS URL dispatch (~100-300ms), pops focus briefly, and requires us to register a custom URL handler and correlate responses. Not great in a per-keystroke hot path.
- **Direct SQLite read** (`~/Library/Group Containers/9K33E3U3T4.net.shinyfrog.bear/.../database.sqlite`, column `ZSFNOTE.ZTEXT`). True markdown, fast query (~5-10ms). Drawbacks: not live — Bear writes on a timer/autosave, so right after a paste the DB may be stale; file-locking risk; schema can shift between Bear versions. Unreliable as a real-time signal.

## Better fix options (ranked)

1. **Hybrid: eventtap-as-signal + observer-as-writer.** Add a lightweight non-intercepting `hs.eventtap` that stamps a `recentCmdV` timestamp when it sees ⌘V. Observer fires → also check "was ⌘V within last 500ms?" Rock-solid for ⌘V path. **Drawback:** breaks Paste app (unless it synthesizes ⌘V — quick to test) and drag-and-drop.
2. **Inserted-slice position check.** Diff `last_value` vs `new_value`, locate the new ￼'s byte position. For real paste it equals `lastCaret`; for Enter-extending-blockquote the ￼ is at a line-start position that's *not* lastCaret. Catches Repro 1 cleanly; Repro 2 is trickier because cmd+left already moved caret to line-start.
3. **Stacked: current clipCount gate + keep a small cooldown on fires near a just-typed `>`/`-`/`[` etc.** Tracks the last typed char in the same eventtap; if it was a format-trigger char in the last 100ms, suppress.
