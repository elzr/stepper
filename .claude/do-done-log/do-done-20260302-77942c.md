## Claude:
Here's a summary of what was built:

**The problem**: Your 4 identical LG HDR 4K monitors get their UUIDs scrambled by macOS periodically (on wake, reconnection, etc). Since Lunar identifies displays by UUID, the position names you set (←Left, ⊙Middle Center, Right→, ↑Top Center) end up on the wrong monitors.

**The fix** (two parts):

1. **Immediate**: Fixed the one missing name — the top monitor (UUID `1FE47444...`) was showing as "LG HDR 4K" instead of "↑Top Center". Lunar was restarted to pick it up.

2. **Automatic going forward**: When all 5 displays are detected (transitioning from fewer), the system now:
   - Waits for screens to stabilize (2s debounce + 3s extra)
   - Uses `screenswitch.buildScreenMap()` to determine which physical position each screen is in (by frame coordinates relative to the built-in display)
   - Maps each screen's current UUID to the correct position name
   - Runs `scripts/lunar-sync-names.py` to update Lunar's plist
   - Only quits and relaunches Lunar if names actually changed (avoids unnecessary restarts)

**Manual trigger**: You can also run `layout.syncLunarNames()` from the Hammerspoon console at any time.
