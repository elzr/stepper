## Claude:
I prefer JSONC here. These sequences are cryptic — `I → C → O` means nothing without context. Inline comments like `// Menu path: ctrl+opt+I opens Insert menu, C → Columns, O → "Insert 1 column right"` sit right next to the data they describe, which is much better than a `"comment"` property that's loaded into memory but never used by the code.

The `comment` property approach works but it's noise in the data structure — it gets parsed, allocated, and ignored. Comments are metadata about the config, not part of it.

Both bear-notes and hyper-actions now use JSONC, so it's consistent too.
