## Anomaly 1: Minimized Chrome window ("📘 Diccionario de Fuentes") got unminimized

**Root cause:** `M.save()` uses `hs.window.orderedWindows()` (line 363), which returns only **visible, non-minimized** windows. So the minimized Chrome window was never in the manual save — and indeed it's absent from `window-layout-manual.json`.

But `M.restoreFromJSON()` uses `hs.window.allWindows()` (line 511) to build its candidate list, which **includes minimized windows**. When the restore runs, Tier 3 "index fallback" (line 551-558) matches any unmatched window of the same app. If another saved Chrome entry no longer has a matching title, it falls back to this minimized window and calls `win:setFrame()` on it — which **implicitly unminimizes** it.

**Fix:** Filter out minimized (and hidden) windows from the candidate list in `restoreFromJSON()` and `retryMisses()`.

## Anomaly 2: Figma (hidden by RCMD) became visible

**Same mechanism.** `hs.window.allWindows()` includes windows from hidden apps. When restore calls `win:setFrame()` or `win:focus()` (line 601 — z-order restoration focuses every matched window back-to-front), it **unhides** the app.

**Fix:** Same filter — skip windows whose app `isHidden()`, or at minimum don't `focus()` windows that were hidden.

## Anomaly 3: Bear "_event Boda Lu 11apr2026" got tiny (300x250)

This window is **not in the manual save** at all (it was probably minimized or on a different space). But it IS in the autosave at 300x250. Since the window wasn't in the manual save, the restore shouldn't have touched it. However, the autosave captured it at that tiny size — meaning something else (possibly macOS display reconnection) shrank it, and then autosave recorded that bad state.

The more likely scenario: during restore, the Tier 3 index fallback matched it to a different saved entry with a small frame (like Bear's 53x48 entry or the tiny "Find in page" Chrome entry in the manual save). After restore set it to that size, the next autosave recorded the bad frame.

---

## Proposed fixes

All three stem from the same two issues:

1. **`allWindows()` in restore includes minimized/hidden windows** — these should be filtered out
2. **Tier 3 index fallback is too aggressive** — it matches any window of the same app regardless of title similarity, which can apply the wrong frame to the wrong window
