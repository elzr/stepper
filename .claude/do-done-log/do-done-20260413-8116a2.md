Good — those were the only two vulnerable objects. The hotkeys use `hs.hotkey.bind()` which Hammerspoon retains internally, and the other timers/watchers are inside modules that store them properly.

Here's the summary:

**What happened:** Hammerspoon loaded Saturday evening (week 15, correctly). The Monday 00:01 timer and the sleep/wake watcher were both **garbage collected** by Lua because their return values were never saved to a variable. When Monday arrived:
- The 00:01 timer was dead — never fired `updateBearWeeksAsync()`
- The wake handler was dead — never fired on screen wake this morning
- The on-load sync check only runs on reload, and there was no reload

**The fix** (`stepper.lua:985-1005`): Save both objects to `local` variables (`weekTimer`, `sleepWatcher`) so they survive Lua GC for the lifetime of the Hammerspoon session.

**Current state:** Week 16 is active, Hammerspoon reloaded, timer and watcher will persist. Next Monday's automatic update should work.
