## Claude:
Good question. It's not totally harmless. The main issue is in the restore's Phase 2 — z-order restoration:

```lua
-- Phase 2: restore z-order (back-to-front = reverse array order)
for i = #pairs_list, 1, -1 do
  pairs_list[i].win:focus()
end
```

This re-focuses every matched window to rebuild the stacking order. After wake, that means:

1. **Focus disruption** — whatever window you were about to use gets replaced by whatever was frontmost at save time (up to 5 minutes stale)
2. **Visual storm** — every window gets focused in sequence, which flickers across all displays

Other edge cases:
- **Window title changes** — if you switched browser tabs or editor files since the last save, the index fallback could swap same-app windows onto the wrong screens
- **Intentional rearrangement** — if you moved windows in the ~5 minutes since the last auto-save, restore undoes that

So auto-restore without confirmation is risky. The alert approach is better — but we need to fix it actually persisting. Let me check why `math.huge` didn't stick.
